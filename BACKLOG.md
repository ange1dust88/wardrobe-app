# План к бете

Цель — выкатить бету. Ниже только то, что **не готово**. Что сделано — в конце, в «Готово».

## Осталось до беты
- [ ] **Мобилка — визуальная проверка на устройстве.** Код адаптива есть (нижний бар, модалки, тач-меню «⋯» на плитках), но вживую не смотрели: колесо, add-item форма, онбординг. Пройтись руками перед запуском.
- [ ] **Password reset на логине** — Supabase `resetPasswordForEmail` + экран.
- [ ] **Продуктовая аналитика** (Plausible/PostHog) — смотреть, что беты реально делают. Зависимости пока нет.

## Удержание / вау (после базовой беты)
- [ ] **Шаринг аутфита картинкой** — флэт-лей лука → PNG для соцсетей (виральный хук для анонса).
- [ ] **Авто-подбор аутфита** (генератор): семя-вещь → движок собирает лучший комплект по harmony. Крупная фича, после беты.

## Полировка UI
- [ ] **Профиль** — переделать модалку профиля под atelier (вёрстка, поля, состояния сохранения) — сейчас черновая.
- [ ] **Онбординг** — доработать (флоу, вёрстка, копирайт), проверить на мобиле — первое впечатление.
- [ ] **Анимации** — единый проход: переходы между страницами, появление модалок/карточек, вход результатов, ховеры — чтобы не дёргано и консистентно.

## Мелочи / отложено
- [ ] Черновик добавления жив только в памяти сессии (File нельзя сериализовать) — при перезагрузке страницы теряется. Превью теперь не теряется при закрытии/переоткрытии модалки (выводится из File).
- [ ] Тач-меню «⋯» на колесе на мобиле показывается на каждой плитке — при желании заменить на long-press, чтобы не засорять.

## Готово

### Хостинг / хардненинг
- `GET /health`, CORS по `WEB_ORIGIN`, `.env.example` (api+web), error boundary (`error.tsx`+`global-error.tsx`), `DEPLOY.md`.

### Must-have для беты
- Фидбек (кнопка → модалка + `POST /feedback`), аккаунт (экспорт JSON + удаление с каскадом), оптимизация фото (sharp → 1000px webp), мобилка (первый проход).

### Матчинг и гардероб
- Нотис о несочетаемых + «Wear it anyway» + `/how-it-works` (7 осей).
- Разбор скора на ховер («?» → бейджи по осям) — **и на колесе, и в List-виде** (fixed-поповер).
- Матчинг: аудит + ребаланс (fit-ось, 36 достижим, кэш карты по colorType, AVIF, слои).
- List-вид: ленты по категориям + ховер как на колесе.
- Аутфиты: создание, редактирование на колесе, двухколоночная модалка деталей, Duplicate, даты, тосты, dirty-чек, не-плодить-дубли.
- Папки (full-stack), слои, сборка лука переживает переходы между страницами.
- **Исключить вещь из матчинга** (hidden shelf: скрыть с колеса → список справа, вернуть по одной/все, скрыть по сезону/категории).
- **Тач-эквивалент ховер-аффордансов** — меню «⋯» (Edit / Hide / Why this score) на coarse-указателях в колесе и списке.
- Черновик добавления: превью фото стабильно (из File).

### Редизайн и новые экраны (atelier)
- **Atelier-редизайн** целиком: cool-gray палитра, шрифты (Bricolage/Hanken/JetBrains Mono), левый рейл, контекстный топ-стрип, скругления, кастомный лоадер-футболка (GarmentLoader), скелетоны на списке аутфитов.
- **Поиск/фильтр гардероба** — spotlight-поиск (по имени/категории), открывашка в топ-стрипе, Cmd/Ctrl+K.
- **Insights** (аналитика гардероба): стат-бэнд, по категориям, «gathering dust», «Worth adding» с **реальным прогоном через движок** (pairs with N) + «Shop this» ссылки.
- **Should I buy?** (`/test-drive`): ввод вещи → Run → лоадер → вердикт + «Works with / Won't carry» карточками, essential-пробелы, «Add to wardrobe» (префилл формы), клик по карточке → read-only ItemView модалка. Бэк `POST /items/matches/preview`; результат инвалидируется при изменении гардероба.
