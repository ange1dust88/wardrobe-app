# План к бете

Цель — выкатить бету. Ниже только то, что **не готово**, по фазам (приоритет сверху вниз). Что уже сделано — в конце, в «Готово».

## Фаза 0 — Блокеры запуска (хостинг / хардненинг)
Маленькая, но без неё деплоя нет.
- [ ] `GET /health` в API (для аптайм-чеков хостинга).
- [ ] CORS: закрыть `enableCors()` на конкретный origin (env `WEB_ORIGIN`), сейчас открыт всем.
- [ ] `.env.example` для `wardrobe-api` и `wardrobe-web` (все нужные переменные без секретов).
- [ ] Error boundary на фронте — падение не должно давать белый экран (+ фолбэк-экран «что-то пошло не так»).
- [ ] Деплой: API (Render/Railway/Fly) + web (Vercel), prod `NEXT_PUBLIC_API_URL`, README с шагами.

## Фаза 1 — Must-have для беты (in-app)
- [ ] **Фидбек / баг-репорт** — кнопка «Сообщить» + модалка; слать в БД (таблица `Feedback`) или на вебхук/почту. Бетам надо как-то писать.
- [ ] **Аккаунт:** удаление аккаунта (каскад: вещи/аутфиты/папки/фото) + экспорт данных. Храним фото → приватность/доверие. Sign out уже есть.
- [ ] **Мобильная адаптивность** — проход по колесу, нижнему бару, широкой модалке аутфита, шапке на телефоне. Сейчас десктоп-first.
- [ ] **Тумбнейлы фото** на аплоаде (sharp → webp ~400px) — сейчас отдаём фулл-рес, с реальными гардеробами затормозит колесо/список.

## Фаза 2 — Удержание / вау
- [ ] **Шаринг аутфита картинкой** — флэт-лей лука → PNG для соцсетей (виральный хук для анонса).
- [ ] **Исключить вещь из матчинга** без удаления — булево `excluded` на Item (аддитивно), фильтровать в match-map и кандидатах, в гардеробе показывать приглушённо с бейджем, тумблер в карточке.
- [ ] **Тест-драйв вещи** до добавления — прогнать виртуальную вещь через движок против гардероба (сколько метчей, средний балл, лучшие пары, каких слотов нет), не сохраняя. Хук — экран добавления считает превью на лету.
- [ ] **Не плодить дубли аутфитов** — галочка: генератор/сохранение не повторяют уже сохранённый набор; дедуп по сортированному `itemIds`.
- [ ] **Поиск/фильтр гардероба** — когда вещей много (по категории/цвету/сезону).
- [ ] **Аналитика** (Plausible/PostHog) — смотреть, что беты реально делают.

## Фаза 3 — Полировка
- [ ] **Дизайн-проход** в atelier-языке: консистентные тени/скругления/типографика, тёплый фон, ховер/фокус-состояния, дырки в тёмной теме.
- [ ] **Скелетоны/лоадеры** вместо спиннеров на списках.
- [ ] **Авто-подбор аутфита** (генератор): семя-вещь → движок собирает лучший комплект по harmony (ties matching → outfits). Крупная фича, после беты.

## Готово (ветка `feat/conflict-notice`)
- Нотис о несочетаемых вещах + «Wear it anyway» + `/how-it-works` (7 осей).
- Разбор скора на ховер («?» на колесе → бейджи по осям).
- Матчинг: аудит + ребаланс (fit-ось, 36 достижим, кэш карты по colorType, AVIF, слои считаются).
- List-вид: ленты по категориям + ховер как на колесе (эдит-карандаш + обводка метчей).
- Аутфиты: создание, редактирование на колесе, двухколоночная модалка деталей (превью head-to-toe, слои рядом, аксессуары иконками, кольцо-скор, «why it works»-чипы), Duplicate, даты, тосты, dirty-чек.
- Папки (full-stack: таблица + CRUD + фильтр-чипы + назначение на карточке и в модалке).
- Слои (несколько вещей одной категории).
- Сборка лука переживает переходы между страницами.
- UX: подтверждения на удаление, loading/error состояния.
